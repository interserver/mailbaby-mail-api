# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) â€” do not edit manually.

- **[gotcha]** `update-spec.sh` is broken: it tries to `git clone git@github.com:interserver/myadmin-mail-api.git` (inaccessible) and references `public/openapi.yaml` (actual path is `public/spec/openapi.yaml`). To regenerate `public/spec/openapi.json` from the YAML, use a one-liner: `python3 -c "import yaml,json; json.dump(yaml.safe_load(open('public/spec/openapi.yaml')),open('public/spec/openapi.json','w'),indent=2)"`. Do NOT trust the CLAUDE.md claim that `./update-spec.sh` rebuilds the spec until the script is fixed.
- **[gotcha]** Illuminate query builder: passing a closure as an element inside a `$where = [...]` array produces broken SQL like `` `1` = (select * where ...) `` because the closure is treated as a value, not a nested condition. To build an `OR` group, chain `->where(function ($query) { $query->whereNull(...)->orWhere(...); })` directly on the builder instead of appending the closure to the `$where` array. This caused the `Column not found: 1054 Unknown column '1' in 'where clause'` error in `app/controller/Mail.php` log()/count().
- **[pattern]** When the same `$where` filter needs to apply to both a count query and a select query, duplicate the `->where(function(){...})` call on each builder â€” don't try to encode nested boolean groups inside the shared `$where` array.
- **[pattern]** OpenAPI spec edits for client-generator friendliness: to keep API behavior unchanged while fixing 80+ "cannot be used as enum variant name" warnings across Rust/Xojo/Nim/PowerShell/Swift generators, add `x-enum-varnames` alongside numeric-prefixed enum values (e.g. `1h`, `24h`, `7d`) rather than renaming the values themselves. The wire values stay the same; only generated constant names get sane identifiers.
- **[pattern]** In OpenAPI `oneOf`/`anyOf` branches, give each branch a `title:` â€” missing titles cause `[WARN] ModelUtils - Failed to get the schema name: null` across ~66 generators. Applies to `SendMailTo`, `startDate`/`endDate` query schemas, etc.
- **[gotcha]** Responses that only declare `text/plain` (like `/ping`) trigger `No application/json content media type found in response` warnings in ~65 generators. Add a second `application/json` content entry with the same string schema to silence it without changing server behavior.
- **[convention]** Schema property names `body`, `from`, `to`, `type`, `interface`, `data`, `local` are reserved words in one or more target languages (Ada, Nim, PowerShell, Xojo, Rust, Swift). Renaming them is a breaking API change â€” prefer documenting this as expected generator behavior rather than silently renaming in the spec.
- **[env]** Client-generator output logs live in two parallel trees: `mailbaby-api-samples/openapi-output/*.txt` (85 files, OpenAPI Generator 7.x) and `mailbaby-api-samples/swagger-output/*.txt` (41 files, Swagger Codegen). Aggregate warnings across BOTH when auditing spec quality â€” they surface different issues (swagger-output shows `ComposedSchema` type errors the openapi-output doesn't).
- **[env]** `./update-spec.sh` runs `git commit` and `git push` as part of its flow â€” treat it as a publish operation, not a local build step. Don't invoke it casually during exploratory edits.
- **[gotcha]** `php -l` does NOT catch the single-colon-vs-double-colon static-call typo (`Log:info(...)` parses as a goto-style label followed by a function call `info(...)`). These only surface at runtime as `Call to undefined function app\controller\info()`. To audit for them across PHP sources, grep: `\b[A-Z][A-Za-z0-9_]+:[a-zA-Z_]+\s*\(` and filter out `::`, URL schemes (`https?:`, `mailto:`, `ftp:`, etc.), and comment lines. Worth a quick sweep after any edit that touches `Log::`, `Db::`, `Cache::`, `Redis::`, `Request::`, `Response::`.
- **[gotcha]** In Illuminate raw SQL via `Db::raw(...)`, use single-quoted string literals for MySQL string values, not double-quotes. `Db::raw('"subject"')` emits `"subject"` which MySQL treats as an identifier (column name) when `ANSI_QUOTES` mode is on, silently changing semantics. The correct form for a JOIN like `ON h1.field = 'subject'` is `Db::raw("'subject'")`. This bit `app/controller/Mail.php` log() joins on ZoneMTA's headers table.
- **[pattern]** For base64-encoded email attachments, prefer `$mailer->addStringAttachment($decoded, $filename)` over decoding to a tempfile with `tempnam()` + `file_put_contents()` + `addAttachment()` â€” it avoids disk I/O and temp-file cleanup entirely. Also always pass `true` as second arg to `base64_decode($data, true)` so invalid base64 returns `false` instead of silently producing garbage.
- **[pattern]** When accepting user input into a SQL `LIKE` pattern, escape the wildcard metacharacters before interpolating: `str_replace(['\\\\', '%', '_'], ['\\\\\\\\', '\\\\%', '\\\\_'], $userInput)`. Without this, a user-supplied `_` or `%` (e.g. in `messageId`) acts as a wildcard and matches unintended rows.
- **[gotcha]** `app/controller/Mail.php` previously carried webman-scaffold demo methods (`viewtest`, `json`, `file`) that were never routed in `config/route.php`. When auditing a controller that originated from a framework skeleton, cross-check public methods against `config/route.php` â€” unrouted public methods are almost always dead scaffold code and safe to remove.
- **[pattern]** After any non-trivial edit to a PHP file, run `php -l <path>` as a fast syntax gate before assuming the fix is good â€” and remember it's only a syntax check (see the `Log:info` gotcha above for what it misses). For a project-wide sweep use `find app -name '*.php' -exec php -l {} \; 2>&1 | grep -v 'No syntax errors'`.
- **[pattern]** `Db::table(...)->first()` returns `null` when no row matches; always null-check before dereferencing (`$order->mail_id`). Same for `->first('col')` used in helpers like `getMailPassword()` â€” return `null` on miss rather than letting a property access raise a TypeError.
- **[gotcha]** `config/route.php` ends with `Route::disableDefaultRoute()`, so webman's automatic controller-method routing is OFF for this project. Every endpoint must be declared explicitly in `config/route.php` â€” adding a public method to a controller alone does NOT expose it. This is also why scaffold methods like `viewtest`/`json`/`file` were dead code (see other learning).
- **[pattern]** Static files in `public/` are served automatically by webman because `config/static.php` has `enable=true` and `app/middleware/StaticFile.php` is registered â€” so `public/swagger-ui.html` is reachable at `/swagger-ui.html` with no route. BUT a bare `/` request does NOT auto-resolve to `public/index.html`; you must add an explicit `Route::get('/', ...)` (or rely on webman's static file resolver, which only matches when the URL path exactly names a file under `public/`). The static resolver lives in `vendor/workerman/webman-framework/src/App.php::findFile()`.
- **[gotcha]** `.gitignore` lists `/tests/tmp` and `/tests/.phpunit.result.cache` (plural `tests/`), but the project's actual ad-hoc test directory is `test/` (singular). Either name is fine, but if you set up PHPUnit pick one and update `.gitignore` to match â€” otherwise generated PHPUnit caches will be committed.
- **[gotcha]** `.gitignore` lists `/tests/tmp` and `/tests/.phpunit.result.cache` (plural `tests/`), but the project's actual PHPUnit test directory is `test/` (singular). The active config now also ignores `/.phpunit.cache` and `/phpunit.xml` (the local copy of `phpunit.xml.dist`). When adding new test infra, use `test/` and update `.gitignore` to match â€” both names appearing is intentional drift, not duplication.
- **[gotcha]** Unanchored regexes that accept date-like numeric formats will match prefixes of unrelated numeric input. `Mail/Stats::fromTimestamp` originally used `/([0-9]{4})([0-9]{2})([0-9]{2})/` (no `^...$`), so a 10-digit unix timestamp like `1900000000` matched the YYYYMMDD branch on its first 8 chars and silently produced garbage via `mktime()`. Anchor numeric-format regexes with `^...$` and cast `is_numeric` inputs to `(int)` before returning â€” and add a test that proves the fall-through ordering is correct, since `php -l` won't catch this.
- **[pattern]** The OpenAPI YAML at `public/spec/openapi.yaml` uses 8-12+ space indentation for operation-level keys (`summary`, `description`, `operationId`). Plain grep with `^\s{2,4}(operationId|summary):` will miss everything. To enumerate operations, parse the YAML in Python: `python3 -c "import yaml; spec=yaml.safe_load(open('public/spec/openapi.yaml')); [print(m,p,o.get('operationId')) for p,ops in spec['paths'].items() for m,o in ops.items() if m in ['get','post','put','patch','delete']]"`.
- **[reference]** A working OpenAPI-driven MCP server reference implementation lives at `/home/sites/mystage/include/Mcp/{OpenApiParser.php,McpServerFactory.php}` with the entrypoint at `/home/sites/mystage/public_html/mcp/client.php`. Uses the `mcp/sdk` PHP SDK + `symfony/yaml`, caches parsed tool definitions via `var_export` to a PHP file with mtime-based invalidation, and supports X-API-KEY / sessionid / OAuth Bearer auth. - **[gotcha]** `WebFetch https://isitagentready.com/<domain>` returns the page template only â€” the actual scan runs client-side and the per-domain failures/warnings are NOT in the fetched HTML. To audit results you must run the scan in a real browser; fetching the URL just returns the category list and won't tell you what failed for `api.mailbaby.net`.
- **[gotcha]** Editing `composer.json` `autoload`/`autoload-dev` sections and then running `composer install` emits `Warning: The lock file is not up to date with the latest changes in composer.json`. To resync without changing dependency versions, run `composer update --dev <some-dev-pkg>` (e.g. `phpunit/phpunit`) or `composer update --lock`. Don't ignore the warning â€” autoload changes won't be picked up until the lock is refreshed.
