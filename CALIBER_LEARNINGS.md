# Caliber Learnings

Accumulated patterns and anti-patterns from development sessions.
Auto-managed by [caliber](https://github.com/caliber-ai-org/ai-setup) — do not edit manually.

- **[pattern]** In OpenAPI `oneOf`/`anyOf` branches, give each branch a `title:` — missing titles cause `[WARN] ModelUtils - Failed to get the schema name: null` across ~66 generators. Applies to `SendMailTo`, `startDate`/`endDate` query schemas, etc.
- **[gotcha]** Responses that only declare `text/plain` (like `/ping`) trigger `No application/json content media type found in response` warnings in ~65 generators. Add a second `application/json` content entry with the same string schema to silence it without changing server behavior.
- **[convention]** Schema property names `body`, `from`, `to`, `type`, `interface`, `data`, `local` are reserved words in one or more target languages (Ada, Nim, PowerShell, Xojo, Rust, Swift). Renaming them is a breaking API change — prefer documenting this as expected generator behavior rather than silently renaming in the spec.
- **[env]** Client-generator output logs live in two parallel trees: `mailbaby-api-samples/openapi-output/*.txt` (85 files, OpenAPI Generator 7.x) and `mailbaby-api-samples/swagger-output/*.txt` (41 files, Swagger Codegen). Aggregate warnings across BOTH when auditing spec quality — they surface different issues (swagger-output shows `ComposedSchema` type errors the openapi-output doesn't).
- **[env]** `./update-spec.sh` runs `git commit` and `git push` as part of its flow — treat it as a publish operation, not a local build step. Don't invoke it casually during exploratory edits.
- **[gotcha]** `php -l` does NOT catch the single-colon-vs-double-colon static-call typo (`Log:info(...)` parses as a goto-style label followed by a function call `info(...)`). These only surface at runtime as `Call to undefined function app\controller\info()`. To audit for them across PHP sources, grep: `\b[A-Z][A-Za-z0-9_]+:[a-zA-Z_]+\s*\(` and filter out `::`, URL schemes (`https?:`, `mailto:`, `ftp:`, etc.), and comment lines. Worth a quick sweep after any edit that touches `Log::`, `Db::`, `Cache::`, `Redis::`, `Request::`, `Response::`.
- **[gotcha]** In Illuminate raw SQL via `Db::raw(...)`, use single-quoted string literals for MySQL string values, not double-quotes. `Db::raw('"subject"')` emits `"subject"` which MySQL treats as an identifier (column name) when `ANSI_QUOTES` mode is on, silently changing semantics. The correct form for a JOIN like `ON h1.field = 'subject'` is `Db::raw("'subject'")`. This bit `app/controller/Mail.php` log() joins on ZoneMTA's headers table.
- **[pattern]** For base64-encoded email attachments, prefer `$mailer->addStringAttachment($decoded, $filename)` over decoding to a tempfile with `tempnam()` + `file_put_contents()` + `addAttachment()` — it avoids disk I/O and temp-file cleanup entirely. Also always pass `true` as second arg to `base64_decode($data, true)` so invalid base64 returns `false` instead of silently producing garbage.
- **[pattern]** When accepting user input into a SQL `LIKE` pattern, escape the wildcard metacharacters before interpolating: `str_replace(['\\\\', '%', '_'], ['\\\\\\\\', '\\\\%', '\\\\_'], $userInput)`. Without this, a user-supplied `_` or `%` (e.g. in `messageId`) acts as a wildcard and matches unintended rows.
- **[gotcha]** `app/controller/Mail.php` previously carried webman-scaffold demo methods (`viewtest`, `json`, `file`) that were never routed in `config/route.php`. When auditing a controller that originated from a framework skeleton, cross-check public methods against `config/route.php` — unrouted public methods are almost always dead scaffold code and safe to remove.
- **[pattern]** After any non-trivial edit to a PHP file, run `php -l <path>` as a fast syntax gate before assuming the fix is good — and remember it's only a syntax check (see the `Log:info` gotcha above for what it misses). For a project-wide sweep use `find app -name '*.php' -exec php -l {} \; 2>&1 | grep -v 'No syntax errors'`.
- **[pattern]** `Db::table(...)->first()` returns `null` when no row matches; always null-check before dereferencing (`$order->mail_id`). Same for `->first('col')` used in helpers like `getMailPassword()` — return `null` on miss rather than letting a property access raise a TypeError.
- **[gotcha]** `config/route.php` ends with `Route::disableDefaultRoute()`, so webman's automatic controller-method routing is OFF for this project. Every endpoint must be declared explicitly in `config/route.php` — adding a public method to a controller alone does NOT expose it. This is also why scaffold methods like `viewtest`/`json`/`file` were dead code (see other learning).
- **[pattern]** Static files in `public/` are served automatically by webman because `config/static.php` has `enable=true` and `app/middleware/StaticFile.php` is registered — so `public/swagger-ui.html` is reachable at `/swagger-ui.html` with no route. BUT a bare `/` request does NOT auto-resolve to `public/index.html`; you must add an explicit `Route::get('/', ...)` (or rely on webman's static file resolver, which only matches when the URL path exactly names a file under `public/`). The static resolver lives in `vendor/workerman/webman-framework/src/App.php::findFile()`.
- **[gotcha]** `.gitignore` lists `/tests/tmp` and `/tests/.phpunit.result.cache` (plural `tests/`), but the project's actual ad-hoc test directory is `test/` (singular). Either name is fine, but if you set up PHPUnit pick one and update `.gitignore` to match — otherwise generated PHPUnit caches will be committed.
- **[gotcha]** `.gitignore` lists `/tests/tmp` and `/tests/.phpunit.result.cache` (plural `tests/`), but the project's actual PHPUnit test directory is `test/` (singular). The active config now also ignores `/.phpunit.cache` and `/phpunit.xml` (the local copy of `phpunit.xml.dist`). When adding new test infra, use `test/` and update `.gitignore` to match — both names appearing is intentional drift, not duplication.
- **[gotcha]** Unanchored regexes that accept date-like numeric formats will match prefixes of unrelated numeric input. `Mail/Stats::fromTimestamp` originally used `/([0-9]{4})([0-9]{2})([0-9]{2})/` (no `^...$`), so a 10-digit unix timestamp like `1900000000` matched the YYYYMMDD branch on its first 8 chars and silently produced garbage via `mktime()`. Anchor numeric-format regexes with `^...$` and cast `is_numeric` inputs to `(int)` before returning — and add a test that proves the fall-through ordering is correct, since `php -l` won't catch this.
- **[pattern]** The OpenAPI YAML at `public/spec/openapi.yaml` uses 8-12+ space indentation for operation-level keys (`summary`, `description`, `operationId`). Plain grep with `^\s{2,4}(operationId|summary):` will miss everything. To enumerate operations, parse the YAML in Python: `python3 -c "import yaml; spec=yaml.safe_load(open('public/spec/openapi.yaml')); [print(m,p,o.get('operationId')) for p,ops in spec['paths'].items() for m,o in ops.items() if m in ['get','post','put','patch','delete']]"`.

- **[gotcha]** `WebFetch https://isitagentready.com/<domain>` returns the page template only — the actual scan runs client-side and the per-domain failures/warnings are NOT in the fetched HTML. To audit results you must run the scan in a real browser; fetching the URL just returns the category list and won't tell you what failed for `api.mailbaby.net`.
- **[gotcha]** Editing `composer.json` `autoload`/`autoload-dev` sections and then running `composer install` emits `Warning: The lock file is not up to date with the latest changes in composer.json`. To resync without changing dependency versions, run `composer update --dev <some-dev-pkg>` (e.g. `phpunit/phpunit`) or `composer update --lock`. Don't ignore the warning — autoload changes won't be picked up until the lock is refreshed.

- **[pattern]** Run the project's PHPUnit suite with `php vendor/bin/phpunit` — `phpunit.xml.dist` is committed and points at `test/` (singular). Use this rather than `phpunit` from PATH or any tool-launcher; it picks up the right bootstrap (`test/bootstrap.php`) and config.
- **[gotcha]** `app/Mcp/OpenApiParser` writes parsed tool defs to `runtime/mcp/cache/mcp_tools_<md5>.php` and reuses the cache when its mtime is `>=` the spec's mtime. Normal edits to `public/spec/openapi.yaml` invalidate it correctly, but a `git checkout` of an older spec sets mtime to checkout-time and can leave a stale cache that masks the older content. After spec rollbacks (or any time you suspect cached tools are stale), run `rm -f runtime/mcp/cache/*` before re-testing the MCP server.
- **[pattern]** To smoke-test the MCP server end-to-end without starting webman, build a PSR-7 request via `Nyholm\Psr7\Factory\Psr17Factory`, hand it to `new Mcp\Server\Transport\StreamableHttpTransport($req)`, and call `$server->run($transport)`. The required JSON-RPC handshake is: (1) `initialize` → capture `Mcp-Session-Id` response header, (2) `notifications/initialized` (echo the session id back as a header), (3) `tools/list` (also with session id). Without step 2 the server returns errors on subsequent calls.
- **[correction]** **Do NOT run `php start.php` (or restart webman) on this host** — production runs on a different machine. There may be a long-lived local webman process visible via `pgrep -f 'php start.php'` (e.g. PID 2557867) but it is NOT what serves api.mailbaby.net. Verify deployed changes by curling `https://api.mailbaby.net/...` after the user pushes, not by starting webman locally.
- **[gotcha]** `WebFetch https://isitagentready.com/<domain>` returns only the static page template; the actual scan runs client-side and per-domain results are NOT in the fetched HTML. To verify agent-readiness improvements, curl the individual discovery endpoints directly and check status codes / payloads (`/.well-known/mcp/server-card.json`, `/.well-known/mcp.json`, `/.well-known/webmcp`, `/.well-known/agent-skills/index.json`, `/.well-known/oauth-protected-resource`, `/.well-known/api-catalog`, `/robots.txt`, `/llms.txt`, `/sitemap.xml`).
- **[convention]** Agent-readiness scanners check several conflicting/overlapping well-known paths for the MCP server card. This project serves the SAME card JSON at all of: `/.well-known/mcp/server-card.json` (canonical SEP-1649), `/.well-known/mcp.json` (SEP-1960 alias), `/.well-known/mcp` (extension-less alias), and `/.well-known/mcp/server.json` (legacy). Add new aliases as routes in `config/route.php` pointing at `Mcp::serverCard` rather than duplicating the body.
- **[convention]** Agent Skills 0.2.0 (`agentskills.io`) requires SKILL.md frontmatter with `name` (max 64 chars, lowercase + digits + hyphens, no leading/trailing hyphen) and `description` (max 1024 chars, non-empty). Skills live in `app/Mcp/Skills/<slug>.md` and are advertised via `/.well-known/agent-skills/index.json` whose `skills[].digest` field is a `sha256:` hash of the file. Don't put skills under `public/` — they're served by `Mcp::agentSkill` so the routing/digest stays consistent with the index.
- **[gotcha]** WebMCP manifest format is officially `TBD` per the upstream proposal at `webmachinelearning/webmcp` — the well-known path is not formally specified. Current scanners look for `mcpServers` + `tools` at `/.well-known/webmcp` (no extension) or `/.well-known/webmcp.json`. Serve both URLs and include both the proposal's hinted `tools[].{name,description,inputSchema}` shape AND an `mcpServers` block with `{type:'streamable-http',url,headers:{X-API-KEY:'${MAILBABY_API_KEY}'}}` so existing readers all match.
